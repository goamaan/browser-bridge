# Security Policy

## Reporting a vulnerability

Please do not open a public issue for sensitive security reports.

Use GitHub's private vulnerability reporting for this repository if it is enabled. If it is not available, contact the maintainers privately through GitHub and include:

- a clear description of the issue
- affected versions or commit ranges, if known
- reproduction steps or a proof of concept, if available
- impact assessment

## Scope

Security reports are especially useful for:

- live Chrome/CDP attach behavior
- daemon transport and local socket handling
- browser/session persistence
- packaged CLI install/update flows
- skill installation or path-handling logic
- unintended credential, token, or browser-session exposure
